# SYSTEM & WORKFLOW CONFIGURATION: VIDEO CLIPPING AUTOMATION

## 1. Environment Overview
I am building a high-level video automation workflow in **n8n** to convert long-form YouTube videos into viral shorts. My entire stack is self-hosted on **Docker Desktop**.
* **n8n URL:** `http://localhost:5678`
* **Minio (S3 Storage) URL:** `http://host.docker.internal:9000`
* **NCA Toolkit URL:** `http://host.docker.internal:8080`

## 2. Current Workflow Logic
1.  **Trigger:** n8n Form (Input: YouTube URL, Clip Size, Duration, Number of Clips).
2.  **Download:** `media/download` endpoint (NCA Toolkit) -> Saves `.mp4` to local Minio bucket.
3.  **Metadata:** `media/metadata` endpoint -> Fetches video data via Minio URL.
4.  **Database:** Airtable node records metadata, description, and the Minio video URL.
5.  **Transcription:** `media/transcribe` endpoint -> Generates transcript/SRT via Webhook (Resume URL).
6.  **Retrieval:** Fetches SRT and a `segments` URL containing the transcript with timestamps.
7.  **Processing:** A JavaScript node splits the transcript into 24 chunks for AI context limits.
8.  **AI Selection:** An AI model analyzes these chunks to find "viral moments" and returns start/end times.

## 3. The Core Problems
* **Timestamp Accuracy:** The timestamps generated by the AI based on transcript segments are **not accurate**. They drift by several seconds, cutting off speech.
* **Face Detection:** I need a 100% free, local way to detect faces and center the video for vertical 9:16 crops.
